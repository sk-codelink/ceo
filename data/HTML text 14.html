1. System Flow

  1. User → Frontend

    • User enters a prompt (and any metadata, e.g. plan, tier) into your React UI.

  2. Frontend → /api/evo/swarm

    • A single POST to /api/evo/swarm carries { prompt, plan, tier, … }.

  3. /api/evo/swarm Handler

    • Authenticate (if needed): check API key or user session.

    • Route: inspect plan and tier to decide which in-process plugin sequence to run.

    • Execute: call your EvoPlanner core (no Redis, no LangChain) with inline plugins for vision, emotion, LLM, affiliate, etc.

    • LLM Call: use the Claude Opus or GPT HTTP client directly—pass the prompt or composed prompt templates.

    • Return: send back { result: "..." } JSON to the frontend.

  4. Frontend ← Display

    • Show the AI response, any links, or UI updates based on the returned payload.

      ...                 ← Add more plans here
  /plugins
    cameraPlugin.js      ← In-process stubs for each capability

    llmPlugin.js
    affiliatePlugin.js
    // add new plugin.js files here
  /services
    aiClient.js          ← HTTP wrapper for Claude Opus / OpenAI
    // any other shared helper (e.g. auth.js)
  /components
    PromptInput.js       ← UI for entering prompts
    ResultDisplay.js     ← UI for showing AI responses
    // other React components
/public
  // static assets if any

  • /pages/api/evo/swarm.js
    – The single entrypoint.
    – Dispatches to evoPlanner.js with the right plan from /lib/plans/*.json.

  • /lib/evoPlanner.js
    – Loads a plan JSON: an ordered list of steps ([{ type: "camera" }, …]).
    – Maps each type to one of your plugins in /plugins/.
    – Passes a shared context object through them.

  • /plugins/*.js
    – Each exports an async execute(params, context) function.
    – E.g. visionPlugin could call Claude Opus Vision, llmPlugin makes the text API call.

  • /services/aiClient.js
    – Simple wrapper over Axios or fetch for your LLM calls.
    – Central place for API keys and timeouts.

  • /components
    – Pure React; no extra API routes here.

3. Function Count

With only /pages/api/evo/swarm.js in your api folder, you have:

  • 1 serverless function → well under the 12-function Free plan cap.

Everything else (plugins, planner logic, service clients) lives in the bundled code for that one function.
